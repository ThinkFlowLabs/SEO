Conventions. Each task has an ID (S1-3 = Sprint 1, task 3), a priority badge (P0 do-first, P1, P2), an effort tag, and a "Done when" acceptance line. The empty checkbox is for you to tick.

Do not jump to building new pages (Sprint 2) before the title and schema fixes ship, or the new pages inherit the same CTR problem. Work the order.

S1-1

Kill the duplicate-brand title bug

codeP0XS
What
Titles on ~24 /areas/ pages render the brand twice: Commercial Cleaning Delray Beach | MB Clean Solutions | MB Clean Solutions.
Why
These are your highest-impression pages (delray 2,200 impr, boynton 1,487, 0.02% CTR). The double brand wastes ~20 SERP chars and reads as low quality.
Root cause
The root app/layout.tsx sets title.template: '%s | MB Clean Solutions', and every metaTitle/metaTitleEs in lib/content/areas.ts ALREADY ends with | MB Clean Solutions. The template appends it a second time.
Fix
In src/lib/content/areas.ts, strip the trailing | MB Clean Solutions and | MB Clean Solutions (ES) from every metaTitle/metaTitleEs. Let the layout template add the brand once. (While here, apply the shorter rewrites from S1-5.)
Done when
View-source on /areas/delray-beach shows exactly one "MB Clean Solutions" in the title, under 60 chars total.
S1-2

Add x-default hreflang

codeP0XS
What
hreflang declares en and es but not x-default, so Google has no fallback for ambiguous searchers.
Fix
In src/lib/seo/metadata.ts, inside alternates.languages (currently lines ~51-54), add one line:
Code
languages: {
  en: alternateEn,
  es: alternateEs,
  'x-default': alternateEn,   // add this
}
Done when
Rendered <head> includes <link rel="alternate" hreflang="x-default" ...> site-wide. One edit fixes every page.
S1-3

Enable Vercel Web Analytics

offsiteP0XS
What
The <Analytics /> component is already mounted (app/[locale]/layout.tsx:63, imports @vercel/analytics/next), but /_vercel/insights/script.js returns 404 on every page, it is collecting nothing.
Why
The code is correct; the feature is just not switched on in the Vercel project. You are flying blind on real-user behavior.
Fix
No code change. Vercel dashboard, MB Clean project, Analytics tab, click Enable Web Analytics. (Optional: also enable Speed Insights.) Redeploy if prompted.
Done when
/_vercel/insights/script.js returns 200 and the Analytics tab shows live pageviews.

codeP1XS
What

Done when
H1 reads ... Commercial Kitchen in South Florida with the space, on all industry pages.

Rewrite homepage + top-20 titles & metas

codeP0M
What

Library
